# AGENTS.md

## Project Overview

115Quick is a Go server for offline downloading via 115 cloud storage API. It accepts magnet links, submits them to 115's offline download service, polls for completion, and downloads finished files locally.

## Build & Run

```bash
go build -o quick.exe .              # Windows
go build -o quick .                  # Linux/Mac
go run quick.go                      # Run directly (default config: etc/quick.yaml)
go run quick.go -f etc/quick.yaml    # Explicit config path
```

No Makefile, no CI, no test files exist.

## Architecture

Framework: **go-zero** (rest server)

```
quick.go (entrypoint)
  └─ internal/config/config.go     (Config struct)
  └─ internal/handler/routes.go    (generated by goctl - DO NOT EDIT)
  └─ internal/handler/Download/*.go (HTTP handlers)
  └─ internal/logic/Download/*.go   (business logic layer)
  └─ internal/svc/servicecontext.go (ServiceContext - core state & download orchestration)
  └─ internal/svc/auth115.go        (115 API client, auth, polling, file download)
  └─ internal/types/types.go        (generated by goctl - DO NOT EDIT)
  └─ internal/types/json_type.go    (hand-written 115 API types)
  └─ internal/respType/             (response wrappers)
```

## Key Patterns

- **goctl generated files**: `internal/handler/routes.go` and `internal/types/types.go` are generated by goctl 1.8.1. Never edit manually; regenerate with goctl if API definitions change.
- **Handler → Logic → Svc**: Standard go-zero pattern. Handlers parse requests, logic layer does business logic, ServiceContext holds shared state.
- **ServiceContext is the brain**: `internal/svc/servicecontext.go` and `auth115.go` contain all core logic (115 auth, offline task submission, polling, file download). Logic layer mostly delegates to ServiceContext.
- **Rate-limited async queue**: Download links go through `DownloadLinkMessageChan` (1M buffer) with a 10s ticker + random jitter before calling 115 API. This avoids rate limiting.
- **Encrypted local storage**: Auth tokens and file info stored encrypted in `etc/secure_data.json` via `SecureStorage`. Key is hardcoded in `servicecontext.go:52`.

## Config

`etc/quick.yaml`:
- `Host`/`Port`: server bind (default 0.0.0.0:8889)
- `SecureStorage.DataPath`: encrypted auth data file
- `Auth115.DownloadPath`: local download directory (default: `data`)

## API Routes

All under prefix `/v1/Download`:
- `POST /api/addDownloadLink` - submit magnet link
- `POST /api/addAuthPw` - set auth password
- `POST /api/StartReName` - trigger file renaming
- `GET /api/getServerInfo` - get server status
- `POST /api/setDownloadMode` - set download mode (0=video only, 1=all)

## Important Quirks

- CORS is wide open (`*` origin) - by design for Chrome extension use.
- On startup, auto-generates QR code for 115 auth and polls device code status every 2s.
- Polling intervals: cloud tasks every 60min, file status check every 10s, download queue every 5s.
- Failed downloads retry after 1 hour. Downloads slower than 1KB/s for 1hr are auto-stopped.
- `DoOnceWithCooldownPoll` uses atomic compare-and-swap with 4min cooldown to avoid concurrent polling.
