# AGENTS.md

## Project Overview

115Quick is a Go server for offline downloading via 115 cloud storage API. It accepts magnet links, submits them to 115's offline download service, polls for completion, and downloads finished files locally.

## Build & Run

```bash
go build -o 115quick.exe .              # Windows
go build -o 115quick .                  # Linux/Mac
./115quick -f etc/quick.yaml            # Run with config
./115quick --version                    # Show version
```

## Architecture

Framework: **go-zero** (rest server)

```
quick.go (entrypoint)
  └─ internal/config/config.go     (Config struct)
  └─ internal/handler/routes.go    (generated by goctl - DO NOT EDIT)
  └─ internal/handler/Download/*.go (HTTP handlers)
  └─ internal/logic/Download/*.go   (business logic layer)
  └─ internal/svc/servicecontext.go (ServiceContext - core state & download orchestration)
  └─ internal/svc/auth115.go        (115 API client, token refresh, polling, file download)
  └─ internal/svc/store.go          (SQLite storage layer)
  └─ internal/types/types.go        (generated by goctl - DO NOT EDIT)
  └─ internal/types/json_type.go    (hand-written 115 API types)
  └─ internal/version/version.go    (version info, GitHub release check)
  └─ internal/respType/             (response wrappers)
  └─ plugin/115Quick/               (Chrome extension source - Manifest V3)
  └─ scripts/                       (macOS install & update scripts)
```

## Key Patterns

- **goctl generated files**: `internal/handler/routes.go` and `internal/types/types.go` are generated by goctl 1.8.1. Never edit manually; regenerate with goctl if API definitions change.
- **Handler → Logic → Svc**: Standard go-zero pattern. Handlers parse requests, logic layer does business logic, ServiceContext holds shared state.
- **SQLite storage**: File info stored in SQLite database (`data/115quick.db`) via `internal/svc/store.go`.
- **Config-based auth**: AccessToken and RefreshToken read from `etc/quick.yaml`, auto-refreshed when expired.
- **Rate-limited async queue**: Download links go through `DownloadLinkMessageChan` (1M buffer) with a 10s ticker + random jitter before calling 115 API.
- **Unified API response**: All API endpoints return `{"code": 200, "data": {...}}` format via `respType.SuccessResp`.

## Config

`etc/quick.yaml`:
- `Host`/`Port`: server bind (default 0.0.0.0:8889)
- `DBPath`: SQLite database path (default: `data/115quick.db`)
- `Auth115.DownloadPath`: local download directory (default: `data`)
- `Auth115.AccessToken`: 115 OAuth access token (optional, can be set via API)
- `Auth115.RefreshToken`: 115 OAuth refresh token (optional, can be set via API)
- `SMB.*`: SMB network storage configuration

## API Routes

All under prefix `/v1/Download`:

### Core
- `POST /api/addDownloadLink` - submit magnet link
- `GET /api/getServerInfo` - get server status (includes version info)
- `POST /api/setDownloadMode` - set download mode (0=video only, 1=all)

### Token Management
- `POST /api/setToken` - set 115 AccessToken and RefreshToken
- `GET /api/getTokenStatus` - get token status (configured, valid, expiresAt)

### Downloads
- `GET /api/getDownloadProgress` - get current download progress
- `GET /api/getCloudTasks` - get cloud download task list
- `GET /api/getTaskHistory` - get task history (paginated)
- `POST /api/removeDownloadTask` - remove pending download task
- `POST /api/refreshTasks` - manually refresh cloud task list

### File Management
- `POST /api/StartReName` - trigger file renaming
- `GET /api/getCloudFiles` - get 115 cloud file list
- `POST /api/createFolder` - create folder in 115
- `POST /api/deleteFile` - delete file/folder
- `POST /api/renameFile` - rename file/folder
- `POST /api/moveFile` - move file/folder

### Settings
- `GET /api/getSMBConfig` - get SMB configuration
- `POST /api/setSMBConfig` - set SMB configuration
- `POST /api/testSMBConnection` - test SMB connection
- `POST /api/setDefaultDownloadDir` - set default download directory
- `GET /api/getDefaultDownloadDir` - get default download directory

### System
- `GET /api/getServerLogs` - get server logs (stdout/stderr)

**Features**:
- Left click opens full management panel in new tab
- Right click context menu: open panel, open settings
- Page magnet link detection with push button
- Server version display with update notification

**Pages**:
- Dashboard: server status, token status, quick add, cloud tasks

- Download List: current downloads and pending queue
- Task History: historical download records
- Settings: download mode, SMB config, server info, rename tools

## Important Quirks

- CORS is wide open (`*` origin) - by design for Chrome extension use.
- Polling intervals: cloud tasks every 60min, file status check every 10s, download queue every 5s.
- Failed downloads retry after 1 hour. Downloads slower than 1KB/s for 1hr are auto-stopped.
- `DoOnceWithCooldownPoll` uses atomic compare-and-swap with 4min cooldown to avoid concurrent polling.
- macOS SMB mount point cannot be `/mnt` (read-only), use user directory like `~/smb`.

## CI/CD

GitHub Actions workflow (`.github/workflows/release.yml`):
- Triggered on version tags (`v*`)
- Builds server binaries for: windows/amd64, linux/amd64, linux/arm64, darwin/amd64, darwin/arm64
- Builds Chrome extension from `plugin/115Quick`
- Creates GitHub Release with all artifacts
- Version injected via `-ldflags` during build

4. Release created with all artifacts
